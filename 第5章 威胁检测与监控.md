# 第5章 威胁检测与监控

## 5.1 威胁检测的技术体系

### 5.1.1 终端检测与响应（EDR）

EDR是终端安全的核心技术，提供终端层面的威胁检测和响应能力。

**EDR核心功能：**
- **行为监控**：监控终端上的进程、文件、网络活动
- **威胁检测**：识别恶意软件、异常行为
- **响应能力**：隔离、清除、取证分析

**EDR架构：**
```
终端Agent → 数据收集 → 云端分析 → 响应处置
```

**主流EDR产品：**
- CrowdStrike Falcon
- SentinelOne
- Microsoft Defender for Endpoint

### 5.1.2 网络检测与响应（NDR）

NDR通过分析网络流量来检测威胁。

**NDR核心功能：**
- **流量分析**：深度包检测和分析
- **异常检测**：识别异常网络行为
- **威胁狩猎**：主动搜索潜在威胁

**NDR工作原理：**
1. 收集网络流量数据
2. 建立正常行为基线
3. 检测偏离基线的异常行为
4. 关联威胁情报

### 5.1.3 安全信息与事件管理（SIEM/SOAR）

SIEM提供安全事件的集中管理和分析能力。

**SIEM核心功能：**
- **日志收集**：从多个来源收集日志
- **事件关联**：关联不同来源的安全事件
- **告警管理**：统一管理和响应告警

**SOAR扩展能力：**
- **自动化响应**：根据预设剧本自动响应
- **编排工作流**：设计复杂的响应流程
- **案例管理**：跟踪安全事件的处理过程

## 5.2 基于威胁情报的主动检测

### 5.2.1 威胁情报的类型

威胁情报分为不同级别：

| 级别 | 描述 | 例子 |
|------|------|------|
| **战术情报** | 具体攻击指标 | IOC（IP、域名、哈希） |
| **操作情报** | 攻击手法和技术 | TTP（技术、战术、程序） |
| **战略情报** | 威胁组织和动机 | APT组织分析 |

### 5.2.2 威胁情报的应用

**情报应用场景：**
- **实时阻断**：根据IOC阻断恶意流量
- **威胁检测**：利用TTP检测攻击行为
- **态势感知**：了解威胁格局和趋势

**情报整合流程：**
1. **情报收集**：从多个渠道获取情报
2. **情报分析**：验证和评估情报质量
3. **情报应用**：将情报导入检测系统
4. **效果评估**：评估情报的实际效果

## 5.3 基于ATT&CK框架的威胁狩猎

### 5.3.1 ATT&CK框架简介

MITRE ATT&CK是一个攻击战术和技术的分类框架。

**ATT&CK矩阵：**
- **战术**：攻击者的目标（如初始访问、执行、持久化）
- **技术**：实现战术的具体方法
- **子技术**：技术的具体变体

### 5.3.2 威胁狩猎流程

威胁狩猎是主动搜索潜在威胁的过程：

**狩猎流程：**
1. **假设生成**：基于威胁情报提出假设
2. **数据收集**：收集相关日志和流量数据
3. **分析验证**：验证假设是否成立
4. **响应处置**：对发现的威胁进行处置
5. **知识沉淀**：将发现反馈到检测规则

## 5.4 告警运营：从海量告警到有效事件

### 5.4.1 告警分诊与优先级判定

面对海量告警，需要进行有效的分诊：

**分诊流程：**
1. **初步筛选**：过滤误报和低优先级告警
2. **优先级判定**：根据风险评估确定优先级
3. **告警聚合**：将相关告警聚合为事件

**优先级判定因素：**
- 资产重要性
- 威胁等级
- 攻击阶段

### 5.4.2 告警富化与上下文关联

告警富化可以提供更完整的上下文信息：

**富化信息来源：**
- 资产信息
- 用户信息
- 威胁情报
- 历史事件

**上下文关联价值：**
- 理解攻击的完整链条
- 评估攻击的真实影响
- 支持准确的响应决策

## 5.5 安全编排自动化与响应（SOAR）实战

### 5.5.1 SOAR的核心能力

**自动化响应：**
- **剧本编排**：定义响应流程
- **自动执行**：无需人工干预
- **自适应响应**：根据情况调整策略

**SOAR工作流示例：**
```
告警触发 → 验证确认 → 自动隔离 → 通知人员 → 日志取证
```

### 5.5.2 实战案例

**案例：勒索软件响应**
1. 检测到勒索软件活动
2. 自动隔离受感染终端
3. 阻止横向移动
4. 通知安全团队
5. 启动取证流程

## 5.6 【AI深度进阶】AI驱动的智能检测体系

### 5.6.1 从规则引擎到AI模型：检测范式的根本变革

传统检测依赖规则引擎，存在局限性：

| 维度 | 规则引擎 | AI模型 |
|------|---------|--------|
| **检测能力** | 已知威胁 | 未知威胁 |
| **适应性** | 静态规则 | 动态学习 |
| **误报率** | 较高 | 较低 |
| **扩展性** | 有限 | 强 |

### 5.6.2 基于机器学习的异常行为检测与用户行为分析

**机器学习检测方法：**
- **监督学习**：使用标注数据训练模型
- **无监督学习**：发现未知模式
- **半监督学习**：结合标注和未标注数据

**用户行为分析（UBA）：**
- 建立用户正常行为基线
- 检测异常行为
- 识别内部威胁

### 5.6.3 Transformer模型在零日恶意软件检测中的应用

Transformer模型在恶意软件检测中展现出强大能力：

**应用场景：**
- **恶意代码检测**：分析代码特征
- **恶意文档检测**：分析文档内容
- **网络流量分析**：识别恶意流量模式

**技术优势：**
- 处理长序列数据
- 捕捉上下文关系
- 迁移学习能力

### 5.6.4 告警分诊智能体：自动化降噪与优先级判定

**告警分诊智能体功能：**
- **智能降噪**：过滤误报和低价值告警
- **优先级判定**：基于AI模型评估风险
- **告警聚合**：将相关告警合并为事件
- **响应建议**：生成最优响应策略

### 5.6.5 威胁狩猎智能体：从被动响应到主动猎杀

**威胁狩猎智能体能力：**
- **假设生成**：基于威胁情报生成狩猎假设
- **自动搜索**：在数据湖中搜索证据
- **模式识别**：发现隐藏的威胁模式
- **报告生成**：自动生成狩猎报告

## 5.7 本章小结

本章介绍了威胁检测与监控的核心技术，包括EDR、NDR、SIEM/SOAR等。特别强调了AI在威胁检测中的深度应用，展示了从规则引擎到AI模型的范式转变，以及告警分诊智能体和威胁狩猎智能体的能力。

**关键要点：**
- 威胁检测需要多层次技术体系的协同
- 威胁情报和ATT&CK框架是主动防御的重要工具
- 告警运营需要有效的分诊和上下文关联
- AI正在从根本上改变威胁检测的方式
