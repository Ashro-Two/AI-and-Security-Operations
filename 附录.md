#### E.4.2 Python脚本示例（资产查询与防火墙封禁）

```python
import requests
import json

def block_malicious_ip(ip_address, reason):
    """
    封禁恶意IP并创建工单
    :param ip_address: 要封禁的IP地址
    :param reason: 封禁原因（如暴力破解）
    """
    
    # Step 1: 从CMDB查询资产信息（CMDB=配置管理数据库）
    asset_info = query_asset(ip_address)
    print(f"查询到资产: {asset_info['hostname']}")
    
    # Step 2: 调用防火墙API封禁IP
    # duration=3600 表示封禁1小时，可根据需求调整
    firewall_response = requests.post(
        "https://firewall/api/v1/block",  # 替换为实际防火墙API地址
        headers={"Authorization": "Bearer YOUR_FIREWALL_TOKEN"},  # 替换为实际Token
        json={"ip": ip_address, "duration": 3600}
    )
    
    # Step 3: 在工单系统创建事件工单
    ticket_data = {
        "title": f"自动封禁IP: {ip_address}",
        "description": f"原因: {reason}\n资产: {asset_info['hostname']}",
        "priority": "High"
    }
    requests.post(
        "https://jira/api/v1/issues",  # 替换为实际工单系统地址
        headers={"Authorization": "Bearer YOUR_JIRA_TOKEN"},  # 替换为实际Token
        json=ticket_data
    )

def query_asset(ip_address):
    """
    从CMDB查询资产信息
    :param ip_address: 资产IP地址
    :return: 资产信息字典
    """
    response = requests.get(
        f"https://cmdb/api/v1/assets?ip={ip_address}",  # 替换为实际CMDB地址
        headers={"Authorization": "Bearer YOUR_CMDB_TOKEN"}  # 替换为实际Token
    )
    return response.json()["results"][0]

# Usage
block_malicious_ip("192.168.1.100", "Brute force attempt")
```

### E.5 Sigma规则示例

#### E.5.1 Brute Force Detection

```yaml
title: SSH Brute Force Attempt
logsource: sshd
detection:
  selection:
    EventID: 4688
    CommandLine|contains: 'sshd'
  filter:
    IpAddress|startswith: '10.'
  condition: selection and not filter
falsepositives:
  - Legitimate admin activity
level: High
```

#### E.5.2 Suspicious PowerShell Execution

```yaml
title: Suspicious PowerShell Download
logsource: powershell
detection:
  selection:
    ScriptBlockText|contains:
      - 'Invoke-WebRequest'
      - 'System.Net.WebClient'
      - '.DownloadFile'
  condition: selection
falsepositives:
  - Legitimate script execution
level: Medium
```

### E.6 SIEM查询语法示例

#### E.6.1 Splunk SPL查询（横向移动检测）

```spl
index=security EventCode=4688 
| stats count BY src_ip, dest_ip, ProcessName 
| where count > 3 AND ProcessName="psexec.exe" 
| lookup geoip src_ip AS src_ip OUTPUT country 
| eval severity = if(count > 10, "HIGH", "MEDIUM") 
| table _time, src_ip, dest_ip, ProcessName, count, country, severity
```

#### E.6.2 Kusto KQL查询（多源关联）

```kusto
SecurityEvent
| where EventID == 4688
| join kind=inner (
    DeviceNetworkEvents
    | where ActionType == "ConnectionSuccess"
) on DeviceName
| where TimeGenerated between (ago(1h)..now())
| summarize Count = count() by SourceIpAddress, DestinationIpAddress
| where Count > 5
| project SourceIpAddress, DestinationIpAddress, Count
```
